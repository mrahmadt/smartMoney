# Admin Panel

SmartMoney includes a comprehensive admin panel built with Filament v5 for managing all configuration.

## Access

Admin resources are restricted to `user_id === 1`. Regular users only see the Dashboard, Transactions, Review Categories, and Alerts pages.

## Resources

### Accounts

Manage the mapping between Firefly III asset accounts and local configuration.

| Field | Description |
|-------|-------------|
| Account Name | Firefly III account display name (synced) |
| Currency | Account's primary currency code |
| User | Which user owns this account |
| Sender | Which SMS sender (bank) this account belongs to |
| Shortcodes | JSON array of account number fragments for SMS matching |
| Budget ID | Firefly III budget for transactions on this account |

**Sync:** The account list auto-syncs from Firefly III. Local accounts not found in Firefly are deleted.

### Categories

Manage spending categories synced from Firefly III.

| Field | Description |
|-------|-------------|
| Name | Category name (unique, synced from Firefly III) |
| Enable Prompt | Whether to include category-specific AI prompt |
| Category Prompt | Custom prompt text sent to the AI when detecting this category (max 200 chars) |

**Sync:** Categories are synced from Firefly III on the Category Mappings page load. New categories are created; existing ones are never deleted.

### Category Mappings

Map merchant/store names to categories.

| Field | Description |
|-------|-------------|
| Merchant Name | The store/merchant name as it appears in SMS |
| Category | Primary spending category |
| Alternative Categories | Optional additional categories for review |

**AI Suggest:** Click the sparkle icon to have AI suggest alternative categories based on the merchant name and existing mappings.

### SMS Messages

View all received SMS messages (admin only).

| Field | Description |
|-------|-------------|
| Sender | Bank name |
| Message | Raw SMS text |
| Is Valid | Whether the SMS passed validation |
| Is Processed | Whether a transaction was created |
| Errors | Any processing errors |

### SMS Regular Expressions

Manage regex patterns for parsing bank SMS.

| Field | Description |
|-------|-------------|
| Sender | Which bank this pattern applies to |
| Transaction Type | withdrawal/deposit/payment/transfer |
| Regular Expression | PHP regex with named groups |
| Strip New Lines | Pre-process option |
| Is Active | Enable/disable this pattern |
| Is Valid | Whether this is a valid transaction pattern (vs. rejection pattern) |

Patterns are auto-generated by the AI agent and can be manually edited or disabled.

### SMS Senders

Register bank SMS sender names.

| Field | Description |
|-------|-------------|
| Sender | Bank SMS sender name (e.g., "ALRAJHI", "ALINMA") |
| Is Active | Enable/disable processing for this sender |

### Keywords

SMS cleaning rules for pre-processing.

| Field | Description |
|-------|-------------|
| Keyword | The pattern to match/remove |
| Is Regular Expression | Whether the keyword is a regex pattern |
| Keyword Type | phone, passcode, date, url, or other |
| Is Active | Enable/disable this rule |

### Settings

Key-value configuration store.

| Field | Description |
|-------|-------------|
| Key | Setting identifier |
| Value | Setting value (string) |
| Description | Human-readable description |

All settings are cached for 1 hour. Changes take effect after cache expiry or `php artisan cache:clear`.

### Users

User management with authentication settings.

| Field | Description |
|-------|-------------|
| Name | Display name |
| Email | Login email |
| Password | Authentication password |
| Budget | Firefly III budget assignment |
| Language | UI language (en/ar) |
| Alert via Email | Enable email notifications |
| MFA Required | Enforce two-factor authentication |

## Settings Reference

### SMS Parsing

| Key | Default | Description |
|-----|---------|-------------|
| `parsesms_enabled` | `true` | Master switch for SMS processing |
| `parsesms_regex_enabled` | `true` | Enable regex-based parsing |
| `parsesms_failback_ai` | `true` | Use AI when regex fails |
| `parsesms_failback_detect_category_ai` | `true` | Use AI for category detection |
| `parsesms_model` | `gpt-5-mini` | LLM model for SMS parsing |
| `parsesms_category_model` | `gpt-5-mini` | LLM model for categorization |
| `parsesms_min_sms_length` | `30` | Minimum SMS length to process |
| `parsesms_store_invalid_sms` | `true` | Store invalid SMS for debugging |
| `parsesms_prompt` | (built-in) | Full system prompt for SMS parser |
| `category_prompt` | (built-in) | Full system prompt for categorizer |

### Anomaly Detection

| Key | Default | Description |
|-----|---------|-------------|
| `average_transactions_months` | `3` | Months of history for averages |
| `abnormal_threshold_percentage_destination` | `20` | Threshold % per destination |
| `abnormal_threshold_percentage_category` | `20` | Threshold % per category |
| `abnormal_threshold_percentage_destination_min` | `50` | Min absolute difference (destination) |
| `abnormal_threshold_percentage_category_min` | `100` | Min absolute difference (category) |
| `abnormal_frequency_multiplier` | `2` | Daily count >= Nx average to alert |

### Subscription Detection

| Key | Default | Description |
|-----|---------|-------------|
| `SubscriptionDetector_enabled` | `true` | Enable/disable detector |
| `SubscriptionDetector_go_back_days` | `120` | Days of history to scan |
| `SubscriptionDetector_transactions_recurring_types` | `daily,weekly,...` | Pattern types to detect |
| `SubscriptionDetector_min_amount` | `10` | Minimum amount threshold |

### Cleanup

| Key | Default | Description |
|-----|---------|-------------|
| `cleanup_sms_days` | `30` | Days to keep processed SMS |
| `cleanup_alerts_days` | `30` | Days to keep read alerts |
| `cleanup_transaction_alerts_days` | `5` | Days to keep transaction alerts |

### Notifications

| Key | Default | Description |
|-----|---------|-------------|
| `alert_batch_delay` | `5` | Seconds before sending batched notifications |
